## Output Format

Return ONLY the raw JSON object with no markdown fences, no explanation, no code blocks. Just the JSON text starting with { and ending with }.

{
  "root": "<root node name>",
  "nodes": [
    { "name": "<unique>", "type": "<operator>", "init": { ... }, "inputs": ["<dep1>", "<dep2>"] }
  ]
}

Rules:
- "root" receives external data. All other nodes reference it or each other.
- "inputs" are dependency paths. Use dot notation for nested access: "tick.close".
- Nodes with no inputs (e.g. Const) are auto-triggered by root.
- No cycles allowed. The graph must be a DAG.
- Each node's input count must match its operator's expected parameters.
- "init" is optional. Use constructor params matching the operator's init spec.

Example — EMA crossover:
{
  "root": "bar",
  "nodes": [
    { "name": "fast", "type": "EMA", "init": { "period": 12 }, "inputs": ["bar.close"] },
    { "name": "slow", "type": "SMA", "init": { "period": 26 }, "inputs": ["bar.close"] },
    { "name": "diff", "type": "Sub", "inputs": ["fast", "slow"] }
  ]
}
